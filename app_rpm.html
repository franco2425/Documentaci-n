CU08 – Asignación de Frecuencias de Despacho

Objetivo

Permitir al despachador asignar, modificar y administrar las frecuencias de salida de las unidades, asegurando que cada bus tenga un horario de partida coherente con la programación y demanda operativa.

Actores

  • Despachador

Flujo Principal

  1. El despachador abre el módulo “Frecuencias de Despacho”.
  2. El sistema muestra:
    • Unidades activas disponibles para asignación.
    • Horarios actuales y próximos intervalos.
    • Indicadores visuales según demanda (verde-normal, amarillo-ajustada, rojo-saturado).
  3. El despachador selecciona una unidad o ruta.
  4. El sistema muestra un selector táctil de hora y minuto (time picker).
  5. El despachador selecciona la hora de salida deseada.
  6. El sistema sugiere:
    • Intervalo recomendado según flujo.
    • Conflictos de horario si existen.
  7. El despachador confirma la frecuencia.
  8. El sistema registra y sincroniza la programación en tiempo real.

Flujos Alternos

  1. FA-01: Si la frecuencia seleccionada se superpone → alerta y sugerencia de horario alterno.
  2. FA-02: Si la unidad está en mantenimiento → bloqueo.
  3. FA-03: Si no existen unidades disponibles → sugerencia de reasignación.

Reglas de Negocio

IDRegla
RN201No se pueden programar dos salidas de la misma unidad dentro del mismo intervalo mínimo.
RN202Las frecuencias deben sincronizarse con la planificación diaria.
RN203El despachador puede modificar horarios solo dentro de su turno asignado.
CU09 – Gestión de la Cola de Despacho

Objetivo

Administrar la cola de unidades en espera de salida, registrando la hora de inicio de operación y controlando prioridades según orden de llegada y estado operativo de cada unidad.

Actores

  • Despachador

Contexto Operativo

La cola de despacho permite registrar la salida de la unidad, marcando el inicio de su operación en ruta. Indicadores clave:

  • Círculo Rojo: El conductor no tiene logueo GPS.
  • Enter para despachar rápidamente (movil → botón “Despachar”).
  • Ingreso rápido de padrón (ej. 005) en un campo numérico.

Flujo Principal

  1. El despachador abre “Cola de Despacho”.
  2. El sistema lista:
    • Unidades en cola.
    • Prioridad de menor a mayor como lo define la operación.
    • Indicadores de GPS, estado, conductor y hora estimada.
  3. El despachador selecciona una unidad o ingresa el padrón en un campo táctil numérico.
  4. El sistema valida:
    • Si el conductor tiene o no sesión GPS activa.
    • Si la unidad no está marcada como fuera de servicio.
  5. El sistema muestra la hora actual y un botón “Registrar Salida”.
  6. El despachador confirma.
  7. El sistema registra:
    • Hora exacta de salida.
    • Inicio de operación.
    • Actualización de frecuencia y ruta.
  8. La unidad desaparece de la cola y aparece en “En operación”.

Flujos Alternos

  1. FA-01: No hay logueo GPS → advertencia, pero permite despachar.
  2. FA-02: Unidad fuera de servicio → bloqueo.
  3. FA-03: Unidad ya fue despachada → refresco de lista.

Reglas de Negocio

IDRegla
RN301Las unidades se ordenan por prioridad operativa definida por la operación.
RN302El sistema debe permitir despachar unidades incluso sin GPS, pero anotando incidencia.
RN303No se puede registrar salida si la unidad está marcada como fuera de servicio.
RN304El registro de salida debe generar una marca de tiempo exacta y no editable.
